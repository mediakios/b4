# B4J Coding Standard

- Use explicit and descriptive variable names.
- Keep Subs focused.
- Avoid hidden shared state.
- Initialize Lists and Maps before use.
- Use Boolean returns for operations that can fail predictably.
- Return initialized empty collections instead of Null.
- Include necessary helper Subs.
- Do not use `Object` as a project field name; use `Obyek`.
- Avoid creating `Main.bas`.
